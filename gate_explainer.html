NoiseBridge · Component ①

Noise-Aware Attention Gate

✕ Close
Purpose

The noise-aware attention gate learns to suppress tokens that have been corrupted by ASR noise before pooling the sequence into a sentence representation. Instead of treating all tokens equally, it assigns each token a scalar weight in (0, 1) based on how trustworthy it is.

Intuition: When ASR corrupts "madarchod" → "madrachd", the corrupted token lands in an unfamiliar region of the embedding space. The gate learns that such tokens are less reliable and down-weights their contribution to the final sentence vector.

Equations

Gate computation

\[ g_\psi(\mathbf{H}) = \mathbf{H} \odot \sigma\!\left(\mathbf{W}_g \mathbf{H} + \mathbf{b}_g\right) \]
Equation (1) — element-wise gating

Gated mean-pooling

\[ \mathbf{z} = \frac{\displaystyle\sum_{i=1}^{L} g_\psi(\mathbf{h}_i) \cdot m_i}{\displaystyle\sum_{i=1}^{L} m_i} \]
Equation (2) — attention-masked pooling

The gate adds only d+1 parameters (768 weights + 1 bias) to the model — negligible overhead.

Key Finding
Ablation result (XLM-R, seed 42)

Gate alone: avg F1 = 0.8339 vs. baseline 0.8394 (−0.55 F1). Adding PWNIC: avg F1 = 0.8401 (+0.62 recovery). Adding Aux head: 0.8434 (+0.33 further).

Without PWNIC, the gate is harmful

The gate has no signal about which tokens are noisy. Without PWNIC contrastive alignment, it learns to suppress tokens arbitrarily — including clean informative ones — reducing performance below baseline.
